Skills are organized into bucket folders under `skills/`:

- `engineering/` — daily code work

Every skill in `engineering/` must have a reference in the top-level `README.md` and an entry in `.claude-plugin/plugin.json`.

Each skill entry in the top-level `README.md` must link the skill name to its `SKILL.md`.

The bucket folder has a `README.md` that lists every skill in the bucket with a one-line description, with the skill name linked to its `SKILL.md`.

## Git

Committing directly on `main` is fine in this repo — the global "never work on main" rule does not apply here. Use descriptive commit messages (e.g. `kira-style-implementation: …`); the `GRLD-` ticket prefix is for the erebor/Jira project, not this repo.
